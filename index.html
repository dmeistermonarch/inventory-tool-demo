On-hand of record
The on-hand of record for every item at this location. "Start count session" opens the phone flow to walk around and update quantities — only items you count change, and each posts its variance to the ledger.
Catalog · per-location pars
Quick edit unlocks par/reorder (for the selected location) plus search term & preferred vendor (which apply to all locations) right in the table. Edit on a row opens the full item detail. Item settings are shared across locations; only par & reorder differ per location.
Locations
Warehouses & storage for this brand. New locations copy par targets from a template location; on-hand starts at 0. Site admins add + deactivate; full removal is Monarch Admin only.
Suppliers
Master list is Monarch-managed (add/remove = Monarch Admin only). Site teams don't add suppliers — they activate the ones they buy from and set their own lead times (Amazon ≈ 2 wks to Hawaii). A1 can sit on the list while a site simply leaves it inactive. Lead times drive expected delivery + the stale flag.
Users & access
One role per user per brand; roles nest — Field/HK ⊂ Receiver ⊂ Purchaser ⊂ Site Admin ⊂ Monarch Admin (+ read-only FP&A Viewer). Purchaser includes receiving and owns catalog/locks; Site Admin adds people & locations for their brands; Monarch Admin is platform-wide. Google SSO for Monarch staff; external-email supported (Eden Crest). Enforcement lands with Stoic IdP.
Viewing as Monarch Admin — all users, all brands. A Site Admin opens this same page and sees only users with access to their brands (and only those access rows).
Brands
Each brand gets its own catalog, locations, and theming. Use the switcher (top right) to change context. Adding brands is Monarch Admin only.
